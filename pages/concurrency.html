⚡ 多线程与并发

并发编程让 Java 应用充分利用多核 CPU。理解线程模型和并发工具是后端开发的核心技能。

创建线程

// 方式一：继承 Thread
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("线程运行: " + getName());
    }
}
new MyThread().start();

// 方式二：实现 Runnable（推荐）
Runnable task = () -> System.out.println("Runnable 运行");
new Thread(task).start();

// 方式三：Callable + Future（有返回值）
Callable<Integer> callable = () -> {
    Thread.sleep(1000);
    return 42;
};
FutureTask<Integer> ft = new FutureTask<>(callable);
new Thread(ft).start();
Integer result = ft.get();   // 阻塞直到拿到结果

线程池（ExecutorService）

永远不要直接用 new Thread()，使用线程池管理线程资源：

import java.util.concurrent.*;

// 创建线程池
ExecutorService pool = Executors.newFixedThreadPool(4);

// 提交任务
pool.execute(() -> System.out.println("无返回值任务"));

Future<String> future = pool.submit(() -> {
    Thread.sleep(500);
    return "完成";
});
String result = future.get(3, TimeUnit.SECONDS);  // 超时控制

// 关闭线程池
pool.shutdown();                      // 不再接受新任务，等待已有任务完成
pool.awaitTermination(30, TimeUnit.SECONDS);
// pool.shutdownNow();               // 立即停止

线程池参数

// 手动创建线程池，明确参数
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    2,                                  // corePoolSize — 核心线程数
    8,                                  // maximumPoolSize — 最大线程数
    60, TimeUnit.SECONDS,               // 空闲线程存活时间
    new LinkedBlockingQueue<>(100),     // 工作队列容量
    Executors.defaultThreadFactory(),   // 线程工厂
    new ThreadPoolExecutor.AbortPolicy() // 拒绝策略（默认抛异常）
);

// 拒绝策略：
// AbortPolicy — 抛 RejectedExecutionException
// CallerRunsPolicy — 调用者线程执行
// DiscardPolicy — 丢弃不抛出异常
// DiscardOldestPolicy — 丢弃队列中最旧的任务

线程安全与锁

// synchronized — 隐式锁
public class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}

// ReentrantLock — 显式锁（更灵活）
Lock lock = new ReentrantLock();
lock.lock();
try {
    // 临界区代码
} finally {
    lock.unlock();   // 必须 finally 中释放
}

// ReadWriteLock — 读多写少场景优化
ReadWriteLock rw = new ReentrantReadWriteLock();
rw.readLock().lock();   // 多个线程可同时读
rw.writeLock().lock();  // 写时独占

并发工具类

类用途
CountDownLatch等待 N 个线程都完成后再继续
CyclicBarrierN 个线程相互等待，全部到达后再继续
Semaphore控制同时访问的线程数（限流）
CompletableFuture异步编排，链式组合多个异步任务
// CountDownLatch — 等待所有线程完成
CountDownLatch latch = new CountDownLatch(3);
ExecutorService pool = Executors.newFixedThreadPool(3);
for (int i = 0; i < 3; i++) {
    pool.submit(() -> {
        try { Thread.sleep(1000); }
        catch (InterruptedException ignored) {}
        latch.countDown();   // 完成一个，计数减一
    });
}
latch.await();   // 主线程等待所有任务完成
System.out.println("全部完成");

// CompletableFuture — 异步编排
CompletableFuture
    .supplyAsync(() -> fetchUser())
    .thenApply(user -> user.getOrders())
    .thenAccept(orders -> System.out.println(orders))
    .exceptionally(e -> {
        log.error("查询失败", e);
        return null;
    });

volatile 与原子类

import java.util.concurrent.atomic.*;

// volatile — 保证可见性，不保证原子性
private volatile boolean running = true;

// Atomic 类 — 无锁线程安全
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();              // ++i
count.getAndIncrement();              // i++
count.compareAndSet(1, 100);          // CAS 操作

AtomicLongArray arr = new AtomicLongArray(10);
arr.incrementAndGet(0);

AtomicReference<String> ref = new AtomicReference<>("初始");
ref.compareAndSet("初始", "新值");
⚠️ 常见陷阱
  • 不要使用 new Thread()，始终用线程池
  • 避免使用 Executors.newCachedThreadPool()（无上限线程）
  • synchronized 不要锁住非 final 字段
  • 线程池的 submit(Runnable) 会吞掉异常，需要用 Future.get() 捕获
  • 锁的范围尽可能小，避免死锁
