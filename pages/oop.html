🏗️ 类与对象

🎯 本节学习目标

  • 掌握类定义、构造方法、getter/setter 的写法
  • 理解接口和抽象类的区别及各自适用场景
  • 掌握方法重写（Override）和方法重载（Overload）的区别

定义类与创建对象

public class Car {
    // 字段
    private String brand;
    private int year;

    // 构造方法
    public Car(String brand, int year) {
        this.brand = brand;
        this.year = year;
    }

    // getter / setter
    public String getBrand() { return brand; }
    public void setBrand(String brand) { this.brand = brand; }

    // 行为方法
    public void honk() {
        System.out.println(brand + " 鸣笛！");
    }
}

// 使用
Car myCar = new Car("Tesla", 2024);
myCar.honk();              // Tesla 鸣笛！
myCar.setBrand("BMW");
System.out.println(myCar.getBrand());  // BMW

封装

使用 private 隐藏实现细节，通过 public 方法暴露接口：

public class BankAccount {
    private double balance;   // 外部不可直接访问

    public BankAccount(double initial) {
        if (initial >= 0)
            this.balance = initial;
    }

    public double getBalance() { return balance; }

    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public boolean withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }
}

继承

public class Animal {
    protected String name;

    public Animal(String name) { this.name = name; }

    public void speak() {
        System.out.println(name + " 发出声音");
    }
}

public class Dog extends Animal {
    public Dog(String name) {
        super(name);     // 调用父类构造方法
    }

    @Override
    public void speak() {
        System.out.println(name + " 汪汪！");
    }
}

// 使用
Animal a = new Dog("Buddy");
a.speak();                     // Buddy 汪汪！

接口

接口定义契约，实现类必须实现所有抽象方法：

public interface Flyable {
    void fly();  // 默认 public abstract
}

public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟儿飞翔");
    }
}

// 接口作为类型
Flyable f = new Bird();
f.fly();

抽象类

public abstract class Shape {
    public abstract double area();   // 子类必须实现

    public void printArea() {         // 普通方法
        System.out.println("面积: " + area());
    }
}

public class Circle extends Shape {
    private double r;

    public Circle(double r) { this.r = r; }

    @Override
    public double area() { return Math.PI * r * r; }
}
⚠️ 初学者常见错误
  • 混淆重载与重写：重载（Overload）是同一个类中方法名相同参数不同；重写（Override）是子类重新定义父类方法
  • 继承滥用：优先使用组合（has-a）而非继承（is-a），继承增加了类之间的耦合
  • 构造方法调用可重写方法：在构造方法中调用可被子类重写的方法可能导致意料之外的行为
  • 接口与抽象类选错：需要共享状态用抽象类，只需定义契约用接口
  • 访问修饰符使用不当：字段默认用 private，需要子类访问用 protected

Record 类 Java 14+

Record 是用于不可变数据载体的特殊类，自动生成构造方法、getter、equals、hashCode、toString：

public record Point(int x, int y) { }
// 等价于手动编写：private final fields + 全参构造 + getter + equals + hashCode + toString

// 使用
Point p = new Point(3, 4);
System.out.println(p.x());     // 3（注意：getter 没有 get 前缀）
System.out.println(p);          // Point[x=3, y=4]
System.out.println(p.equals(new Point(3, 4)));  // true
💡 何时使用 Record？ 当类的主要作用是承载不可变数据时（DTO、值对象、多返回值），Record 是最佳选择。
📌 本节要点
  • 封装：用 private 隐藏实现，通过 public 方法控制访问
  • 继承：用 extends 复用代码，子类通过 super 调用父类构造方法
  • 多态：父类引用指向子类对象，调用重写的方法时执行子类的版本
  • 接口定义契约（can-do），抽象类提供公共基础（is-a），Record 简化不可变数据类
