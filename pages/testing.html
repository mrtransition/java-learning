🧪 单元测试

🎯 本节学习目标
  • 掌握 JUnit 5 的核心注解：@Test、@BeforeEach、@BeforeAll
  • 学会使用 Mockito 隔离外部依赖
  • 学会使用 MockMvc 测试 Controller 层
  • 理解测试金字塔和不同层级的测试策略

测试是保障代码质量的基石。企业级项目通常包含单元测试、集成测试和端到端测试三层。

JUnit 5 基础

import static org.junit.jupiter.api.Assertions.*;

import org.junit.jupiter.api.*;

class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @Test
    void shouldAddTwoNumbers() {
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    void shouldThrowOnDivisionByZero() {
        assertThrows(ArithmeticException.class,
            () -> calculator.divide(10, 0));
    }

    @Test
    void shouldCheckString() {
        assertAll("grouped assertions",
            () -> assertTrue("hello".startsWith("h")),
            () -> assertEquals(5, "hello".length()),
            () -> assertNotNull("hello")
        );
    }
}

Mockito — 隔离外部依赖

import static org.mockito.Mockito.*;
import org.mockito.junit.jupiter.MockitoExtension;

@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @InjectMocks
    private UserService userService;

    @Test
    void shouldFindUserById() {
        User mockUser = new User("Alice", "alice@example.com");
        when(userRepository.findById(1L)).thenReturn(Optional.of(mockUser));

        User result = userService.findById(1L);

        assertNotNull(result);
        assertEquals("Alice", result.getName());
        verify(userRepository).findById(1L);   // 验证方法被调用
    }

    @Test
    void shouldThrowWhenUserNotFound() {
        when(userRepository.findById(anyLong())).thenReturn(Optional.empty());

        assertThrows(ResourceNotFoundException.class,
            () -> userService.findById(99L));
    }

    @Test
    void shouldVerifyInteraction() {
        userService.delete(1L);
        verify(userRepository).deleteById(1L);
        verifyNoMoreInteractions(userRepository);
    }
}

MockMvc — 测试 Controller

import java.util.Arrays;

@SpringBootTest
@AutoConfigureMockMvc
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private UserService userService;

    @Test
    void shouldReturnUserList() throws Exception {
        when(userService.findAll(any(Pageable.class)))
            .thenReturn(Arrays.asList(new User("Alice", "alice@test.com")));

        mockMvc.perform(get("/api/v1/users")
                .accept(MediaType.APPLICATION_JSON))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$", hasSize(1)))
            .andExpect(jsonPath("$[0].name").value("Alice"));
    }

    @Test
    void shouldReturn404WhenUserNotFound() throws Exception {
        when(userService.findById(99L))
            .thenThrow(new ResourceNotFoundException("用户不存在"));

        mockMvc.perform(get("/api/v1/users/99"))
            .andExpect(status().isNotFound())
            .andExpect(jsonPath("$.message").value("用户不存在"));
    }
}

Testcontainers — 真实数据库集成测试

import org.testcontainers.containers.MySQLContainer;

@SpringBootTest
@Testcontainers
class UserRepositoryTest {

    // 启动真实 MySQL 容器
    @Container
    static MySQLContainer<?> mysql = new MySQLContainer<>("mysql:8.0")
            .withDatabaseName("testdb");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", mysql::getJdbcUrl);
        registry.add("spring.datasource.username", mysql::getUsername);
        registry.add("spring.datasource.password", mysql::getPassword);
    }

    @Autowired
    private UserRepository userRepository;

    @Test
    void shouldSaveAndFindUser() {
        User user = new User("Alice", "alice@test.com");
        user = userRepository.save(user);

        Optional<User> found = userRepository.findById(user.getId());
        assertTrue(found.isPresent());
        assertEquals("Alice", found.get().getName());
    }
}

测试金字塔与策略

层级速度比例工具
单元测试毫秒级70%JUnit + Mockito
集成测试秒级20%@SpringBootTest + Testcontainers
端到端测试分钟级10%Cypress / Playwright
💡 测试最佳实践
  • 方法名使用 shouldXxxWhenYyy 格式，清晰表达意图
  • 遵循 AAA 模式：Arrange（准备）→ Act（执行）→ Assert（断言）
  • 每个测试只测一个行为，一个断言失败不应影响其他测试
  • 使用 @ParameterizedTest 测试多组数据
  • 集成测试用 @Testcontainers 而非 H2 内存库
⚠️ 初学者常见错误
  • 测试方法不是 public：JUnit 5 允许 default/package 可见性，但不要用 private
  • 测试间共享状态：每个测试应是独立的，不应依赖执行顺序
  • Mock 太多：过度 Mock 导致测试失去意义，集成测试用真实依赖
  • 只测正确路径：别忘了测试边界情况和异常分支
  • 忽略测试可读性：方法名用 shouldXxxWhenYyy 格式，遵循 AAA 模式
📌 本节要点
  • JUnit 5 提供 @Test、断言、参数化测试等功能
  • Mockito 通过 @Mock + @InjectMocks 隔离外部依赖
  • MockMvc 模拟 HTTP 请求测试 Controller 响应
  • Testcontainers 在 Docker 容器中运行真实数据库做集成测试
  • 测试金字塔：70% 单元测试 + 20% 集成测试 + 10% E2E 测试
