📦 集合框架

Java Collections Framework 提供了一套统一的接口和实现类。核心接口：

🎯 本节学习目标
  • 掌握 List、Set、Map、Queue 四大接口的特点和选用场景
  • 理解 ArrayList vs LinkedList、HashMap vs TreeMap 的区别
  • 学会使用 Collections 工具类进行排序、查找、不可变包装
  • 了解不可变集合（Java 9+ List.of / Set.of / Map.of）
接口特点常用实现
List有序、可重复ArrayList, LinkedList
Set不可重复HashSet, TreeSet, LinkedHashSet
Map键值对HashMap, TreeMap, LinkedHashMap
Queue队列（FIFO）ArrayDeque, LinkedList

List

List<String> fruits = new ArrayList<>();

fruits.add("Apple");
fruits.add("Banana");
fruits.add("Cherry");
fruits.add("Apple");    // List 允许重复

System.out.println(fruits.get(0));        // Apple
System.out.println(fruits.size());         // 4
System.out.println(fruits.contains("Banana"));  // true

// 遍历
for (String fruit : fruits) {
    System.out.println(fruit);
}

Set

Set<Integer> numbers = new HashSet<>();

numbers.add(3);
numbers.add(1);
numbers.add(2);
numbers.add(3);   // 重复元素被忽略

System.out.println(numbers.size());     // 3
System.out.println(numbers.contains(2));  // true

// HashSet 无序，如需有序：
Set<Integer> sorted = new TreeSet<>(numbers);
Set<Integer> insertionOrder = new LinkedHashSet<>(numbers);

Map

Map<String, Integer> scores = new HashMap<>();

scores.put("Alice", 95);
scores.put("Bob", 87);
scores.put("Charlie", 92);

System.out.println(scores.get("Alice"));       // 95
System.out.println(scores.getOrDefault("David", 0));  // 0

// 遍历
for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

// Java 8+ 便捷方法
scores.merge("Bob", 5, Integer::sum);  // Bob 变为 92
scores.computeIfAbsent("Eve", k -> 88);  // 键不存在时计算

集合排序与比较

List<String> names = Arrays.asList("Charlie", "Alice", "Bob");

// 自然排序（元素需实现 Comparable）
Collections.sort(names);                         // [Alice, Bob, Charlie]

// 自定义排序（Comparator）
names.sort((a, b) -> b.compareTo(a));            // [Charlie, Bob, Alice]
names.sort(Comparator.comparingInt(String::length)); // [Bob, Alice, Charlie]

// 稳定排序：sorted() 保留相等元素的相对顺序
Collections.reverse(names);
Collections.shuffle(names);                      // 随机打乱

HashMap 与 hashCode 契约

HashMap 依赖 hashCode() 和 equals() 定位和比较键。作为键的类必须遵守：

  • 如果 a.equals(b) 为 true，则 a.hashCode() == b.hashCode() 必须成立
  • 如果 hashCode() 相同但 equals() 不同，称为哈希碰撞，链表/红黑树处理
  • 常用作键的类型：String、Integer、Long 等不可变类（已正确实现 hashCode + equals）

不可变集合 Java 9+

// List.of / Set.of / Map.of — 创建不可变集合
List<String> fixed = List.of("x", "y", "z");
Set<Integer> nums = Set.of(1, 2, 3);
Map<String, Integer> ages = Map.of("Alice", 30, "Bob", 25);

// 不可变集合的注意事项
// fixed.add("w");         // ❌ 运行时 UnsupportedOperationException
// 不可变集合不允许 null 元素
⚠️ 初学者常见错误
  • 遍历时修改集合：for-each 遍历 List 时调用 remove() 抛 ConcurrentModificationException，应使用 iterator.remove()
  • HashMap 键使用可变对象：键的 hashCode 改变后无法再被查找
  • 混淆 Comparable 和 Comparator：Comparable 是"内部比较"（类自身实现），Comparator 是"外部比较"（单独定义排序规则）
  • Arrays.asList 返回固定大小 List：返回的 List 不能 add/remove，但可以 set
  • HashSet 无序：需要插入顺序用 LinkedHashSet，需要排序用 TreeSet
📌 本节要点
  • List 有序可重复，Set 不可重复，Map 键值对，Queue FIFO
  • ArrayList 基于数组（随机访问快），LinkedList 基于链表（插入删除快）
  • HashMap 用 hashCode + equals 定位键，作为键的类必须正确实现这两个方法
  • Collections 提供排序、查找、不可变包装等工具方法
