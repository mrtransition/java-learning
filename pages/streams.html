🌊 Lambda 与 Stream API

🎯 本节学习目标
  • 理解 Lambda 表达式语法及其与传统匿名类的区别
  • 掌握四大函数式接口：Predicate、Function、Consumer、Supplier
  • 熟练使用 Stream API 进行过滤、映射、归约操作
  • 掌握 Collectors 分组和分区操作
  • 学会用 Optional 优雅处理 null

Lambda 表达式

Lambda 提供简洁的匿名函数语法，核心格式：(参数) -> 表达式

// 传统匿名内部类
Runnable r1 = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};

// Lambda 等价写法
Runnable r2 = () -> System.out.println("Hello");

// 带参数的 Lambda
Comparator<String> cmp = (a, b) -> a.length() - b.length();

List<String> names = Arrays.asList("Java", "Python", "C");
names.sort((a, b) -> Integer.compare(a.length(), b.length()));
System.out.println(names);  // [C, Java, Python]

函数式接口

Lambda 配合函数式接口（只有一个抽象方法）使用：

接口输入输出示例
Predicate<T>Tbooleans -> s.isEmpty()
Function<T,R>TRs -> s.length()
Consumer<T>Tvoidx -> System.out.println(x)
Supplier<T>无T() -> Math.random()
// 自定义函数式接口
@FunctionalInterface
interface Calculator {
    int calculate(int x, int y);
}

Calculator add = (a, b) -> a + b;
Calculator multiply = (a, b) -> a * b;

System.out.println(add.calculate(3, 4));       // 7
System.out.println(multiply.calculate(3, 4));  // 12

Stream API

Stream 用于对集合进行声明式处理（过滤、映射、归约等）。

创建 Stream

List<String> items = Arrays.asList("apple", "banana", "cherry");
// List.of(...) (Java 9+) 可替代 Arrays.asList

// 从集合创建
Stream<String> stream1 = items.stream();

// 从数组创建
Stream<Integer> stream2 = Arrays.stream(new Integer[]{1, 2, 3});

// 直接创建
Stream<String> stream3 = Stream.of("a", "b", "c");

// 无限流
Stream<Double> randoms = Stream.generate(Math::random);
Stream<Integer> natural = Stream.iterate(0, n -> n + 1);

中间操作（返回新 Stream）

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6);

List<Integer> result = numbers.stream()
    .filter(n -> n % 2 == 0)           // 过滤偶数: [2, 4, 6]
    .map(n -> n * n)                    // 平方: [4, 16, 36]
    .sorted((a, b) -> b - a)           // 降序: [36, 16, 4]
    .limit(2)                           // 取前2: [36, 16]
    .collect(Collectors.toList());

System.out.println(result);  // [36, 16]

Collectors 分组与分区

groupingBy 按条件分组，partitioningBy 按 boolean 分区——这是 SQL 式 GROUP BY 的 Java 实现：

List<String> words = Arrays.asList("apple", "banana", "cherry", "avocado");

// 按首字母分组
Map<Character, List<String>> byFirst = words.stream()
    .collect(Collectors.groupingBy(w -> w.charAt(0)));
System.out.println(byFirst);
// {a=[apple, avocado], b=[banana], c=[cherry]}

// 分区：长度 > 5 的为一组，其余为另一组
Map<Boolean, List<String>> partitioned = words.stream()
    .collect(Collectors.partitioningBy(w -> w.length() > 5));
System.out.println(partitioned);
// {false=[apple, cherry], true=[banana, avocado]}

// 分组后计数
Map<Character, Long> countByFirst = words.stream()
    .collect(Collectors.groupingBy(w -> w.charAt(0), Collectors.counting()));
System.out.println(countByFirst);  // {a=2, b=1, c=1}

// 分组后映射
Map<Character, List<Integer>> lengthByFirst = words.stream()
    .collect(Collectors.groupingBy(w -> w.charAt(0),
             Collectors.mapping(String::length, Collectors.toList())));
// {a=[5, 7], b=[6], c=[6]}

终端操作

List<String> words = Arrays.asList("hello", "world", "java");

// forEach — 遍历
words.stream().forEach(System.out::println);

// collect — 收集到集合
List<String> upper = words.stream()
    .map(String::toUpperCase)
    .collect(Collectors.toList());     // [HELLO, WORLD, JAVA]

// reduce — 归约
int sum = Arrays.asList(1, 2, 3, 4).stream()
    .reduce(0, Integer::sum);          // 10

// 统计
long count = words.stream()
    .filter(s -> s.length() > 3)
    .count();                          // 3

boolean any = words.stream()
    .anyMatch(s -> s.startsWith("h"));  // true

Optional

Optional 用于优雅处理可能为 null 的值，避免 NullPointerException：

// 创建
Optional<String> empty = Optional.empty();
Optional<String> present = Optional.of("hello");
Optional<String> nullable = Optional.ofNullable(mightBeNull);

// 安全使用
String result = present
    .map(String::toUpperCase)
    .filter(s -> s.length() > 2)
    .orElse("default");          // "HELLO"

// 链式调用
String name = findUser(userId)
    .flatMap(User::getMiddleName)
    .orElseThrow(() -> new NoSuchElementException("无中间名"));
⚠️ 注意
  • Stream 是惰性的——没有终端操作时中间操作不会执行
  • Stream 只能消费一次，重复使用会抛异常
  • 避免在 Lambda 中修改外部变量——变量应实际上是 final 的
⚠️ 初学者常见错误
  • Stream 只能消费一次：调用终端操作后 Stream 已关闭，再次使用抛 IllegalStateException
  • 忘记终端操作：Stream 是惰性的，没有终端操作（collect/forEach/count）时中间操作不会执行
  • 在 Lambda 中修改外部变量：Lambda 引用的局部变量必须是 effectively final 的
  • 过度使用 Stream：简单 for-each 用传统循环更清晰，Stream 适合复杂数据管道
  • Optional.get() 滥用：用 orElse/orElseThrow 替代直接 get()，避免 NoSuchElementException
📌 本节要点
  • Lambda 语法：(参数) -> 表达式，配合函数式接口使用
  • Stream 操作分中间操作（懒执行）和终端操作（触发执行）
  • 核心函数式接口：Predicate（断言）、Function（转换）、Consumer（消费）、Supplier（供给）
  • Collectors.groupingBy / partitioningBy 实现类似 SQL 的分组统计
  • Optional 避免 NPE，用 orElse/orElseThrow 安全取值
