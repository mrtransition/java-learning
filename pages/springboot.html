🍃 Spring Boot

🎯 本节学习目标
  • 理解 Spring Boot 自动配置原理
  • 掌握 Starter 依赖的使用方法
  • 掌握依赖注入的多种方式及最佳实践
  • 学会使用 application.yml 进行配置管理

Spring Boot 是当前 Java 企业开发的事实标准，通过自动配置大幅简化了 Spring 应用的搭建。

⚠️ 版本说明 本教程示例基于 Spring Boot 2.x（兼容 Java 8）。Spring Boot 3.x Java 17+ 需升级到 JDK 17，请注意区别。

项目初始化

// 启动类 — 入口点
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

@SpringBootApplication 是一个组合注解，等价于：

  • @Configuration — 声明配置类
  • @EnableAutoConfiguration — 启用自动配置
  • @ComponentScan — 扫描当前包及其子包的组件

Starter 依赖

Spring Boot 提供预配置的 Starter 依赖，开箱即用：

Starter用途
spring-boot-starter-web构建 REST API（内嵌 Tomcat）
spring-boot-starter-data-jpaJPA + Hibernate 数据库访问
spring-boot-starter-validationBean Validation (Hibernate Validator)
spring-boot-starter-testJUnit 5 + Mockito + MockMvc
spring-boot-starter-securitySpring Security 认证授权
spring-boot-starter-actuator生产监控端点

自动配置原理

Spring Boot 通过 @Conditional 系列注解实现智能自动配置：

// 只有当 classpath 中存在某个类时，配置才生效
@Configuration
@ConditionalOnClass(DataSource.class)
public class DataSourceAutoConfiguration {
    @Bean
    @ConditionalOnMissingBean
    public DataSource dataSource() {
        return new HikariDataSource();
    }
}

常用条件注解：@ConditionalOnClass、@ConditionalOnMissingBean、@ConditionalOnProperty、@ConditionalOnExpression

依赖注入

@Service
public class UserService {
    private final UserRepository userRepository;

    // 构造器注入（推荐）
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

// 多个实现时用 @Qualifier 指定
@Service
public class OrderService {
    private final PaymentService paymentService;

    public OrderService(@Qualifier("alipay") PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}
💡 构造器注入 vs @Autowired
  • 构造器注入：不可变、便于测试、显式依赖——官方推荐
  • @Autowired 字段注入：简洁但难以测试和发现依赖
  • 使用 final 字段 + 构造器注入是最佳实践

常用注解

注解位置说明
@Controller / @RestController类声明 Web 控制器
@Service类声明业务逻辑层
@Repository类声明数据访问层
@Component类通用 Spring Bean
@Configuration类声明配置类
@Bean方法声明一个 Bean
@Value("${key}")字段/参数注入配置文件属性
@ConfigurationProperties类绑定配置前缀到对象

application.yml vs application.properties

application.properties

server.port=8080
spring.datasource.url=jdbc:h2:mem:testdb
spring.jpa.hibernate.ddl-auto=update
logging.level.com.example=DEBUG

application.yml（推荐）

server:
  port: 8080
spring:
  datasource:
    url: jdbc:h2:mem:testdb
  jpa:
    hibernate:
      ddl-auto: update
logging:
  level:
    com.example: DEBUG
⚠️ 初学者常见错误
  • @SpringBootApplication 位置错误：启动类必须在所有组件类的根包中，否则组件扫描不到
  • @Autowired 字段注入滥用：官方推荐构造器注入（final 字段 + 构造参数），便于测试和不变性
  • 循环依赖：A 依赖 B、B 依赖 A 导致启动失败，应重构代码消除循环
  • 忽略 application.yml 缩进：YAML 对缩进敏感，多级属性缩进不一致导致配置不生效
  • Controller 直接暴露实体：应该用 DTO 和实体解耦，避免序列化问题
📌 本节要点
  • @SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan
  • Starter 依赖提供开箱即用的自动配置
  • 构造器注入是官方推荐的 DI 方式
  • YAML 格式更清晰，Properties 格式更简单，据团队习惯选择
✏️ 动手练习
  1. 创建一个 Spring Boot 项目，包含 spring-boot-starter-web 依赖
  2. 在启动类所在包中创建一个 @Service 和 @RestController
  3. 在 application.yml 中配置 server.port=8081，启动验证
  4. 用 @ConfigurationProperties 绑定一组配置到 Java 对象
  5. 使用 @Value 注入单个配置值并测试
