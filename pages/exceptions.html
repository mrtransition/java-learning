⚠️ 异常处理

🎯 本节学习目标
  • 理解 try/catch/finally 的执行流程
  • 掌握 try-with-resources 自动资源管理
  • 区分受检异常（Checked）和非受检异常（Unchecked）
  • 学会自定义异常及最佳实践

try / catch / finally

import java.io.*;

public class ExceptionDemo {
    public static void main(String[] args) {
        BufferedReader reader = null;
        try {
            reader = new BufferedReader(new FileReader("test.txt"));
            System.out.println(reader.readLine());
        } catch (FileNotFoundException e) {
            System.err.println("文件未找到: " + e.getMessage());
        } catch (IOException e) {
            System.err.println("读取错误: " + e.getMessage());
        } finally {
            if (reader != null) {
                try { reader.close(); }
                catch (IOException ignored) {}
            }
        }
    }
}

try-with-resources Java 7+

自动关闭实现了 AutoCloseable 的资源，无需 finally：

import java.nio.file.*;
import java.util.stream.Stream;

public class TryWithResources {
    public static void main(String[] args) {
        Path path = Paths.get("hello.txt");

        try (Stream<String> lines = Files.lines(path)) {
            lines.forEach(System.out::println);
        } catch (IOException e) {
            System.err.println("读取失败: " + e.getMessage());
        }
    }
}

受检异常 vs 非受检异常

类别说明示例
受检异常 (Checked)必须处理或声明抛出IOException, SQLException
非受检异常 (Unchecked)运行时异常，可忽略NullPointerException, IllegalArgumentException
// 受检异常 — 编译器要求处理
public void readFile(String path) throws IOException {
    // Files.readString (Java 11+) 更简洁
    String content = new String(Files.readAllBytes(Paths.get(path)));
}

// 非受检异常 — 无需声明
public void setAge(int age) {
    if (age < 0)
        throw new IllegalArgumentException("年龄不能为负");
}

自定义异常

// 自定义受检异常
public class InsufficientFundsException extends Exception {
    public InsufficientFundsException(String message) {
        super(message);
    }
}

// 使用
public void withdraw(double amount) throws InsufficientFundsException {
    if (amount > balance)
        throw new InsufficientFundsException("余额不足");
    balance -= amount;
}
💡 最佳实践
  • 用异常处理异常情况，不要用于流程控制
  • catch 最具体的异常类型，避免笼统地捕获 Exception
  • 优先使用 try-with-resources 管理资源
  • 记录或重新抛出异常，不要吞掉
⚠️ 初学者常见错误
  • 捕获过于宽泛的异常：catch (Exception e) 会隐藏预期之外的错误，应捕获最具体的异常类型
  • 吞掉异常：空的 catch 块让问题难以排查，至少记录日志
  • 用异常控制流程：异常处理有性能开销，正常业务流程用 if/else 判断
  • 在 finally 中返回值：finally 中的 return 会覆盖 try/catch 中的 return
  • 资源未关闭：旧代码中 finally 里关闭资源可能为 null，优先用 try-with-resources
📌 本节要点
  • try 监控可能抛异常的代码，catch 处理异常，finally 无论如何都执行（清理资源）
  • try-with-resources 自动关闭 AutoCloseable 资源，代码更简洁安全
  • 受检异常必须处理或声明抛出；非受检异常（RuntimeException）可选处理
  • 自定义异常继承 Exception（受检）或 RuntimeException（非受检），据业务场景选择
✏️ 动手练习
  1. 编写一段代码尝试将字符串 "abc" 转为整数，捕获 NumberFormatException 并输出友好提示
  2. 用 try-with-resources 读取一个文件并打印内容，处理 IOException
  3. 自定义一个 AgeOutOfRangeException（非受检），在设置年龄 < 0 或 > 150 时抛出
  4. 写一个方法演示 finally 块的执行（即使 try 中有 return，finally 也会执行）
