📋 日志与配置管理

🎯 本节学习目标
  • 掌握 SLF4J + Logback 的使用方法
  • 理解日志级别及生产环境的最佳设置
  • 学会多环境配置管理
  • 了解 Actuator 生产监控端点

日志和配置管理是生产环境的基石。好的日志策略能让你在出现问题时快速定位根因。

SLF4J + Logback — 标准日志方案

SLF4J 是日志门面，Logback 是 Spring Boot 默认的日志实现。

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class OrderService {
    private static final Logger log = LoggerFactory.getLogger(OrderService.class);

    public Order createOrder(OrderRequest req) {
        log.info("开始创建订单: userId={}, amount={}", req.getUserId(), req.getAmount());
        try {
            Order order = orderRepository.save(toEntity(req));
            log.info("订单创建成功: orderId={}", order.getId());
            return order;
        } catch (Exception e) {
            log.error("订单创建失败: userId={}", req.getUserId(), e);
            throw new OrderCreateException("创建订单异常", e);
        }
    }
}

日志级别

级别用途
ERROR系统错误，需要立即处理（数据库断开、服务不可用）
WARN潜在问题，不影响运行但需要注意（配置回退、重试）
INFO关键业务节点记录（订单创建、用户注册、支付完成）
DEBUG开发调试信息，生产环境一般关闭
TRACE详细跟踪信息，仅用于深度排查

logback-spring.xml

<!-- src/main/resources/logback-spring.xml -->
<configuration>
    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <!-- 文件输出 — 按天滚动 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/app.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>logs/app.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <!-- 错误日志单独输出 -->
    <appender name="ERROR_FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/error.log</file>
        <filter class="ch.qos.logback.classic.filter.ThresholdFilter">
            <level>ERROR</level>
        </filter>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>logs/error.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>90</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <!-- 按包名设置级别 -->
    <logger name="com.example" level="DEBUG"/>
    <logger name="org.springframework" level="WARN"/>

    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
        <appender-ref ref="ERROR_FILE"/>
    </root>
</configuration>

多环境配置

// application.yml — 公共配置
server:
  port: 8080
spring:
  jpa:
    hibernate:
      ddl-auto: none   # 生产环境用 Flyway 管理

---
// application-dev.yml — 开发环境
spring:
  devtools:
    restart:
      enabled: true
  datasource:
    url: jdbc:h2:mem:testdb
  jpa:
    show-sql: true
logging:
  level:
    com.example: DEBUG

---
// application-prod.yml — 生产环境
spring:
  datasource:
    url: ${DB_URL}
    username: ${DB_USER}
    password: ${DB_PASS}
  jpa:
    show-sql: false
logging:
  level:
    com.example: WARN

配置优先级（从高到低）

  1. 命令行参数：--server.port=8081
  2. 环境变量：JAVA_HOME、SPRING_PROFILES_ACTIVE=prod
  3. 应用级 application-{profile}.yml
  4. application.yml 默认配置

@ConfigurationProperties — 类型安全配置

// application.yml
app:
  oss:
    endpoint: https://oss-cn-hangzhou.aliyuncs.com
    access-key: ${OSS_ACCESS_KEY}
    secret-key: ${OSS_SECRET_KEY}
    bucket: my-app-files

// 配置绑定类
@ConfigurationProperties(prefix = "app.oss")
public class OssProperties {
    private String endpoint;
    private String accessKey;
    private String secretKey;
    private String bucket;
    // getter / setter
}

// 在配置类启用
@Configuration
@EnableConfigurationProperties(OssProperties.class)
public class AppConfig { }

Actuator — 生产监控

// pom.xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

// application.yml 暴露端点
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,env,loggers
  endpoint:
    health:
      show-details: when-authorized

// 访问：GET /actuator/health → 健康检查
// GET /actuator/info → 应用信息
// GET /actuator/metrics → 性能指标
// POST /actuator/loggers/com.example → 动态调整日志级别
💡 日志最佳实践
  • 使用占位符 log.info("user={}", id) 而非字符串拼接
  • 异常日志传入完整异常对象：log.error("消息", exception)
  • 日志中记录业务追踪 ID（traceId）方便关联请求
  • 不要记录密码、身份证号等敏感信息
  • 设置合理的日志保留策略（按磁盘空间和时间）
⚠️ 初学者常见错误
  • 字符串拼接日志：用占位符 log.info("user={}", id) 而非 "user=" + id，避免不必要的字符串构建
  • 日志中记录敏感信息：密码、身份证号、银行卡号等严禁记录
  • 生产环境日志级别过低：生产环境用 INFO/WARN，DEBUG 仅在排查问题时临时开启
  • 日志文件无限增长：必须配置滚动策略（按时间或大小）和保留策略
  • 选择性地记录异常：将完整异常对象传给日志门面：log.error("操作失败", exception)
📌 本节要点
  • SLF4J 是日志门面，Logback 是 Spring Boot 默认实现
  • 日志级别从低到高：TRACE < DEBUG < INFO < WARN < ERROR
  • 多环境配置用 application-{profile}.yml 隔离，生产环境敏感信息用环境变量注入
  • Actuator 提供 /health、/metrics、/info 等生产监控端点
  • @ConfigurationProperties 实现类型安全的配置绑定
✏️ 动手练习
  1. 在 Service 类中添加 SLF4J Logger，分别用 INFO、WARN、ERROR 记录不同级别的日志
  2. 配置 logback-spring.xml：控制台输出 + 按天滚动的文件输出
  3. 创建 application-dev.yml 和 application-prod.yml，配置不同的日志级别
  4. 使用 @ConfigurationProperties 绑定一组 OSS 配置
  5. 启用 Actuator，访问 /actuator/health 和 /actuator/info 端点
