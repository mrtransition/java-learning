📅 日期与时间 API

Java 8 引入全新的 java.time 包，解决旧版 Date / Calendar 设计缺陷。新 API 不可变、线程安全、语义清晰。

LocalDate / LocalTime / LocalDateTime

import java.time.*;

// 当前日期
LocalDate today = LocalDate.now();                     // 2026-06-29
LocalDate specific = LocalDate.of(2026, 6, 29);

// 当前时间
LocalTime now = LocalTime.now();                       // 14:30:00.123
LocalTime noon = LocalTime.of(12, 0);

// 日期 + 时间
LocalDateTime dt = LocalDateTime.now();                // 2026-06-29T14:30
LocalDateTime meeting = LocalDateTime.of(2026, 6, 29, 14, 30);

System.out.println(today.plusDays(7));                 // 2026-07-06
System.out.println(today.minusMonths(1));              // 2026-05-29
System.out.println(dt.getDayOfWeek());                 // MONDAY
System.out.println(dt.getDayOfYear());                 // 180

日期计算与比较

LocalDate start = LocalDate.of(2026, 1, 1);
LocalDate end = LocalDate.of(2026, 12, 31);

Period period = Period.between(start, end);
System.out.println(period.getMonths());   // 11（月差）
System.out.println(period.getDays());     // 30

// 精确时间差
Duration duration = Duration.between(
    LocalDateTime.of(2026, 1, 1, 0, 0),
    LocalDateTime.of(2026, 1, 2, 12, 0));
System.out.println(duration.toHours());   // 36

// 比较
System.out.println(start.isBefore(end));  // true
System.out.println(start.isLeapYear());   // false

格式化与解析

import java.time.format.DateTimeFormatter;

LocalDate date = LocalDate.now();
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("yyyy/MM/dd");
System.out.println(date.format(fmt));               // 2026/06/29

// 解析字符串
LocalDate parsed = LocalDate.parse("2026-06-29");        // ISO 格式默认支持
LocalDate custom = LocalDate.parse("2026/06/29", fmt);

// 常用格式
System.out.println(DateTimeFormatter.ISO_LOCAL_DATE);                     // 2026-06-29
System.out.println(DateTimeFormatter.ISO_LOCAL_DATE_TIME);                // 2026-06-29T14:30:00
System.out.println(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));   // 自定义

时区处理

import java.time.*;

// ZonedDateTime — 带时区的日期时间
ZonedDateTime zdt = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));
System.out.println(zdt);   // 2026-06-29T14:30+08:00[Asia/Shanghai]

// 时区转换
ZonedDateTime ny = zdt.withZoneSameInstant(ZoneId.of("America/New_York"));
System.out.println(ny);    // 2026-06-29T02:30-04:00[America/New_York]

// OffsetDateTime — 固定偏移
OffsetDateTime odt = OffsetDateTime.now(ZoneOffset.ofHours(8));

// 所有可用时区
ZoneId.getAvailableZoneIds().stream()
    .filter(id -> id.contains("Asia"))
    .forEach(System.out::println);

与旧 API 互操作

// Date → Instant → LocalDateTime
import java.util.Date;

Date old = new Date();
Instant instant = old.toInstant();
LocalDateTime ldt = LocalDateTime.ofInstant(instant, ZoneId.systemDefault());

// LocalDateTime → Date
LocalDateTime now = LocalDateTime.now();
Date date = Date.from(now.atZone(ZoneId.systemDefault()).toInstant());

// SQL 日期
LocalDate ld = LocalDate.now();
java.sql.Date sqlDate = java.sql.Date.valueOf(ld);
💡 java.time 最佳实践
  • 业务代码优先用 LocalDate / LocalDateTime，不要用旧 Date
  • API 传输用 ISO 8601 字符串（2026-06-29T14:30:00Z）
  • Jackson 配置 jackson-datatype-jsr310 序列化 java.time
  • 数据库新驱动（JDBC 4.2+）直接支持 LocalDate / LocalDateTime
