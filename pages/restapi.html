🌐 REST API 设计

🎯 本节学习目标
  • 理解 RESTful 资源命名规范
  • 掌握 @RestController CRUD 的完整写法
  • 理解统一响应和全局异常处理的设计模式
  • 学会使用 Bean Validation 校验请求参数

REST API 是企业应用对外交互的核心方式。一个好的 API 设计直接影响系统的可维护性和使用体验。

RESTful 资源命名规范

HTTP 方法路径说明
GET/api/users获取用户列表
GET/api/users/{id}获取单个用户
POST/api/users创建用户
PUT/api/users/{id}全量更新用户
PATCH/api/users/{id}部分更新用户
DELETE/api/users/{id}删除用户
  • 资源名使用复数名词：/users 而非 /user
  • 子资源用嵌套：/users/{id}/orders
  • 操作转译为资源：/POST /payments 而非 /POST /pay
  • 版本号放路径或请求头：/api/v1/users

完整 CRUD Controller

@RestController
@RequestMapping("/api/v1/users")
public class UserController {

    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @GetMapping
    public List<User> list(@RequestParam(defaultValue = "0") int page,
                          @RequestParam(defaultValue = "20") int size) {
        return userService.findAll(PageRequest.of(page, size));
    }

    @GetMapping("/{id}")
    public ResponseEntity<User> getById(@PathVariable Long id) {
        return ResponseEntity.ok(userService.findById(id));
    }

    @PostMapping
    public ResponseEntity<User> create(@Valid @RequestBody User user) {
        User created = userService.create(user);
        URI location = URI.create("/api/v1/users/" + created.getId());
        return ResponseEntity.created(location).body(created);
    }

    @PutMapping("/{id}")
    public ResponseEntity<User> update(@PathVariable Long id,
                                       @Valid @RequestBody User user) {
        return ResponseEntity.ok(userService.update(id, user));
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        userService.delete(id);
        return ResponseEntity.noContent().build();
    }
}

统一响应与全局异常处理

// 统一响应包装
public class ApiResult<T> {
    private int code;
    private String message;
    private T data;

    public static <T> ApiResult<T> success(T data) {
        ApiResult<T> r = new ApiResult<>();
        r.code = 200; r.message = "success"; r.data = data;
        return r;
    }

    public static <T> ApiResult<T> error(int code, String msg) {
        ApiResult<T> r = new ApiResult<>();
        r.code = code; r.message = msg;
        return r;
    }
}

// 全局异常处理
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ApiResult<Void>> handleNotFound(ResourceNotFoundException e) {
        return ResponseEntity.status(404)
                .body(ApiResult.error(404, e.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiResult<Void>> handleValidation(MethodArgumentNotValidException e) {
        String msg = e.getBindingResult().getFieldErrors().stream()
                .map(err -> err.getField() + ": " + err.getDefaultMessage())
                .collect(Collectors.joining("; "));
        return ResponseEntity.badRequest()
                .body(ApiResult.error(400, msg));
    }
}

参数校验

@Data
public class CreateUserRequest {
    @NotBlank(message = "用户名不能为空")
    @Size(min = 2, max = 50)
    private String name;

    @NotBlank
    @Email(message = "邮箱格式不正确")
    private String email;

    @Min(18) @Max(150)
    private int age;

    @Pattern(regexp = "^1[3-9]\\d{9}$", message = "手机号格式错误")
    private String phone;
}

// 分组校验（不同场景不同规则）
public interface CreateGroup {}
public interface UpdateGroup {}

public class UserRequest {
    @Null(groups = CreateGroup.class)
    @NotNull(groups = UpdateGroup.class)
    private Long id;

    @NotBlank(groups = CreateGroup.class)
    private String name;
}

💡 REST API 最佳实践
  • 使用 ResponseEntity.created(location) 返回 201 + Location 头
  • 不要直接暴露实体类——用 DTO（数据传输对象）解耦
  • 分页参数统一使用 page（从 0 开始）+ size
  • 敏感信息（密码、身份证）在序列化时用 @JsonIgnore
  • 全局异常处理避免堆栈泄露到客户端
⚠️ 初学者常见错误
  • 直接暴露实体类：Controller 中返回 Entity 会导致数据泄露（如密码字段），必须用 DTO 解耦
  • 忽略 HTTP 状态码语义：创建返回 201、成功返回 200、未找到返回 404、参数错误返回 400
  • 异常堆栈泄露到客户端：全局异常处理器应返回自定义错误信息而非堆栈信息
  • @PathVariable 和 @RequestParam 混用：路径变更参数用 @PathVariable，查询参数用 @RequestParam
  • 未做参数校验：用 @Valid + @NotBlank/@Email/@Size 等注解进行声明式校验
📌 本节要点
  • REST API 使用名词复数命名资源（/users/{id}），HTTP 方法对应 CRUD 操作
  • @RestController + @RequestMapping 构建控制器
  • 统一响应体 + @RestControllerAdvice 全局异常处理
  • @Valid + Bean Validation 注解实现声明式参数校验
  • 使用 DTO 而非实体类做请求/响应，保护内部数据
