📐 泛型

泛型（Generics）让类型成为参数，编写更通用、类型安全的代码。Java 5 引入，是集合框架的基石。

🎯 本节学习目标
  • 理解泛型解决的问题：类型安全 + 消除强制转型
  • 掌握泛型类、泛型方法的定义
  • 理解通配符 ? extends 和 ? super 及 PECS 原则
  • 了解类型擦除及其带来的限制

为什么需要泛型

// 没有泛型 — 需要强制转型，不安全
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);   // 必须手动转型
list.add(123);                      // 不小心混入 Integer
String s2 = (String) list.get(1);  // ❌ 运行时 ClassCastException

// 有泛型 — 编译器检查类型安全
List<String> safe = new ArrayList<>();
safe.add("hello");
String s3 = safe.get(0);            // 无需转型
// safe.add(123);                   // ❌ 编译错误

泛型类

// 一个可以装任意类型的盒子
public class Box<T> {
    private T value;

    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

// 使用
Box<Integer> intBox = new Box<>();
intBox.set(42);
Integer val = intBox.get();   // 自动拆箱

Box<String> strBox = new Box<>();
strBox.set("Hello");

// 多类型参数
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}

Pair<String, Integer> pair = new Pair<>("age", 30);

泛型方法

// 类型参数放在返回类型之前
public class Utils {
    public static <T> T getMiddle(T... arr) {
        return arr[arr.length / 2];
    }

    public static <T extends Comparable<T>> T max(T a, T b) {
        return a.compareTo(b) > 0 ? a : b;
    }
}

String mid = Utils.getMiddle("a", "b", "c");  // "b"
Integer m = Utils.max(10, 20);                     // 20

通配符（Wildcard）

// ? extends — 生产者，只能读取
public void printAll(List<? extends Number> list) {
    for (Number n : list) {
        System.out.println(n);
    }
    // list.add(42);  // ❌ 编译错误，不能写入
}

// ? super — 消费者，只能写入
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    // Integer i = list.get(0);  // ❌ 编译错误
    Object o = list.get(0);       // 只能当作 Object 读取
}

// 无界通配符
public void printSize(List<?> list) {
    System.out.println(list.size());
}
💡 PECS 原则

Producer Extends, Consumer Super — 生产者用 extends 只读，消费者用 super 只写。

类型擦除

Java 的泛型是编译期实现的，运行时擦除类型参数：

// 编译后，Box<T> 中的 T 被替换为 Object（或上界）
Box<String> b1 = new Box<>();
Box<Integer> b2 = new Box<>();
System.out.println(b1.getClass() == b2.getClass());  // true

// 无法创建泛型数组：new T[] 不允许
// 无法使用 instanceof：if (obj instanceof List<String>) 不允许
⚠️ 初学者常见错误
  • 不能创建泛型数组：new T[] 编译错误，因为类型擦除后运行时不知道 T 的实际类型
  • 不能使用 instanceof：obj instanceof List<String> 编译错误，只能检查 raw type
  • 静态字段不能引用类型参数：private static T value; 不允许，因为静态字段属于类而非实例
  • 基本类型不能作类型参数：Box<int> 错误，必须使用包装类 Box<Integer>
  • 混淆 ? extends 和 ? super：记住 PECS——Producer Extends, Consumer Super
📌 本节要点
  • 泛型在编译期提供类型安全，运行时通过类型擦除实现
  • 泛型类在类名后加 <T>，泛型方法在返回类型前加 <T>
  • 通配符 ? extends T 用于只读场景（生产者），? super T 用于只写场景（消费者）
  • 类型擦除导致无法创建泛型数组、无法 instanceof 泛型类型
