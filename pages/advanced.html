🔧 注解 / IO / 反射

这三个主题贯穿框架底层：注解提供元数据，IO 处理数据读写，反射实现运行时动态操作。

一、注解（Annotation）

注解是附加在代码上的元数据，不影响程序逻辑，但可被编译器或运行时读取处理。

内置注解

注解作用
@Override标记方法重写父类方法
@Deprecated标记已过时的 API
@SuppressWarnings抑制编译器警告
@FunctionalInterface标记函数式接口

自定义注解

import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME)   // 运行时可读
@Target(ElementType.FIELD)           // 作用于字段
public @interface Column {
    String name();
    boolean nullable() default true;
    int length() default 255;
}

// 使用自定义注解
public class User {
    @Column(name = "user_name", nullable = false, length = 50)
    private String username;
}

// 元注解：@Retention（保留策略）、@Target（作用目标）
// RetentionPolicy: SOURCE | CLASS | RUNTIME
// ElementType: FIELD, METHOD, TYPE, PARAMETER, CONSTRUCTOR...

二、Java I/O

文件读写

import java.nio.file.*;
import java.util.stream.Stream;

// Files.readString / writeString (Java 11+) 更简洁
String content = new String(Files.readAllBytes(Paths.get("input.txt")));
Files.write(Paths.get("output.txt"), content.getBytes());

List<String> lines = Files.readAllLines(Paths.get("data.txt"));
Files.write(Paths.get("out.txt"), lines);

// Stream 方式处理大文件
try (Stream<String> stream = Files.lines(Paths.get("large.log"))) {
    stream.filter(line -> line.contains("ERROR"))
          .forEach(System.out::println);
}

// 传统 IO 流
try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"));
     BufferedWriter writer = new BufferedWriter(new FileWriter("out.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        writer.write(line);
        writer.newLine();
    }
}

字节流与缓冲

// 文件复制 — 缓冲字节流
try (InputStream in = new BufferedInputStream(new FileInputStream("source.pdf"));
     OutputStream out = new BufferedOutputStream(new FileOutputStream("copy.pdf"))) {
    byte[] buf = new byte[8192];
    int len;
    while ((len = in.read(buf)) != -1) {
        out.write(buf, 0, len);
    }
}

// NIO 更高效的复制
Files.copy(Paths.get("source.pdf"), Paths.get("copy.pdf"),
           StandardCopyOption.REPLACE_EXISTING);

三、反射（Reflection）

反射让程序在运行时检查或修改类和对象的行为。Spring、Hibernate、JUnit 等框架的核心机制。

获取 Class 对象

// 三种获取方式
Class<?> c1 = String.class;                  // .class 字面量
Class<?> c2 = "hello".getClass();            // 实例的 getClass()
Class<?> c3 = Class.forName("java.lang.String");  // 全限定名

反射操作

import java.lang.reflect.*;
import com.example.Column;

public class ReflectionDemo {
    public static void main(String[] args) throws Exception {
        Class<?> clazz = Class.forName("com.example.User");

        // 创建实例（调用无参构造器）
        Object obj = clazz.getDeclaredConstructor().newInstance();

        // 读取字段值
        Field field = clazz.getDeclaredField("name");
        field.setAccessible(true);              // 绕过 private
        field.set(obj, "Alice");
        System.out.println(field.get(obj));

        // 调用方法
        Method method = clazz.getMethod("getName");
        String name = (String) method.invoke(obj);

        // 获取注解
        Column col = field.getAnnotation(Column.class);
        System.out.println(col.name());

        // 获取泛型信息
        for (TypeVariable t : clazz.getTypeParameters()) {
            System.out.println(t.getName());
        }
    }
}
⚠️ 反射注意事项
  • 反射有性能开销（比直接调用慢 10-100 倍），不要用于高频代码
  • setAccessible(true) 破坏封装性，谨慎使用
  • 反射调用会绕过编译期检查，出错在运行时才暴露
  • 框架层面使用反射，业务代码尽量避免
